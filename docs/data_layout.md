# Data Layout

This page describes source datasets and generated data used by ExPO-HM.

## Directory Structure

- `data/gt/`: source JSON/JSONL metadata and task annotations.
- `data/image/`: source image files for FB, MAMI, and PrideMM.
- `data/verl/`: generated parquet for GRPO training/evaluation.

## Source Metadata (`data/gt/`)
Datasets in open-source scope:

- `FB`
- `MAMI`
- `PrideMM`

Additional annotations shipped in this repo:

- `fine_grained_hateful_memes/`: FB fine-grained protected-class (PC) and attack annotations (used for CDE curriculum / fine-grained evaluation)

Fine-grained SFT uses policy-manual `*_choice` files, registered in:

- `sft/lmfactory_data/dataset_info.sft_release.json`

## Generated Parquet (`data/verl/`)

Produced by `scripts/grpo/data_prep/` scripts.

- Baseline binary parquet: generated by `scripts/grpo/data_prep/generate_hatefulmemes.sh`
- CDE binary + FG parquet: generated by `scripts/grpo/data_prep/generate_cde_all.sh`

Common output subpaths:

- `data/verl/FB/`, `data/verl/MAMI/`, `data/verl/PrideMM/`
- `data/verl/CDE/FB/`, `data/verl/CDE/MAMI/`, `data/verl/CDE/PrideMM/`
- `data/verl/CDE/fb-fine-grained-combined/`
